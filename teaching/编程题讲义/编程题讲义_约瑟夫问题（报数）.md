# 编程题讲义：约瑟夫问题（报数）

> 目标：掌握“循环删除”的经典问题模型：一圈人按固定步长淘汰，问最后剩下谁/输出淘汰顺序。

## 1. 问题模型
有 `n` 个对象围成一圈，从某个位置开始，每次数 `k` 个，数到的人出局，然后从下一个继续。常见要求：
- 输出最后剩下的编号
- 或输出整个出局序列

## 2. 三种常用实现策略（按难度/适用范围）

### 2.1 数组标记法（最直观）
用数组 `alive[i]` 标记是否还在，循环计数，遇到出局就标记为 0。
优点：简单易写。
缺点：`n` 很大时可能慢，但训练题通常够用。

### 2.2 循环链表法（更贴合“删除”）
用链表把人串成环，每次走 `k-1` 步删除下一个。
优点：删除快。
缺点：实现比数组复杂。

### 2.3 递推公式（只求最后结果时最优）
如果只问最后剩下的位置（从 0 编号），有经典递推：
```
f(1) = 0
f(n) = (f(n-1) + k) % n
```
最后答案（从 1 编号）是 `f(n) + 1`。

## 3. 什么时候用哪一种
- 题目要求输出淘汰过程：用数组标记或链表。
- 题目只问最后一个：优先用递推公式。

## 4. 自检清单
- 编号是从 0 还是从 1 开始？输出是否要转换？
- 删除后起点是否移动到“被删的下一个”？
- 循环终止条件是否正确？（只剩 1 个时停）
