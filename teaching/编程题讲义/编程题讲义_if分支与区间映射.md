# 编程题讲义：`if` 分支与区间映射

> 目标：把“分段判断、等级映射、边界处理”写得又快又稳。

## 1. 典型场景
输入一个数（或多个数），输出一个类别/等级/结果，规则通常是“按区间划分”：
- `x < a`
- `a <= x < b`
- `b <= x < c`
- `x >= c`

## 2. 推荐写法：从小到大或从大到小一致
### 2.1 从小到大（最直观）
```c
if (x < a) { ... }
else if (x < b) { ... }     /* 这里隐含 x >= a */
else if (x < c) { ... }     /* 这里隐含 x >= b */
else { ... }                /* 这里隐含 x >= c */
```
优点：条件短，不容易写出重复区间。

### 2.2 从大到小（也可以，但要小心）
```c
if (x >= c) { ... }
else if (x >= b) { ... }
else if (x >= a) { ... }
else { ... }
```

## 3. 边界写法的“统一口径”
建议统一用“左闭右开”：
- `[a, b)` 写成：`x >= a && x < b`

这样所有区间不会重叠，也不会漏。

## 4. 常见坑
- 把 `<=` 写成 `<` 或反过来，导致边界点落错等级。
- 条件顺序写反：例如先判断 `x < c`，后判断 `x < b`，后者永远到不了。
- 忘记 `else`：导致多个分支都执行（如果你没加 `else if`）。

## 5. 自检清单
- 每个输入值是否一定落入某个分支？（有没有漏区间）
- 任何一个输入值是否可能落入两个分支？（有没有重叠）
- 边界点（刚好等于阈值）输出是否符合规则？
