# 编程题讲义：总览与做题流程

> 目标：把“编程题”拆成稳定可复用的套路：读懂输入输出 → 选数据结构 → 选算法模板 → 处理边界与格式。

## 1. 编程题常见能力地图（你需要会哪些）
从基础到进阶，常见知识点可以分为 6 类：
1) **输入输出与格式化**：`scanf/printf`、小数保留位数、字符串读入、行/列读入。
2) **分支与循环**：区间判断、计数循环、嵌套循环、提前结束、循环不变量。
3) **数论与枚举**：因子遍历、素数/完数、水仙花数、最大公因数/最小公倍数、进制转换。
4) **数组、指针与字符串**：`strlen` 指针版、矩阵转置、字符串转整数、反序输出。
5) **递归/回溯**：典型是“摆放/排列/搜索”类问题（需要剪枝）。
6) **链表与结构体**：建立链表、插入/遍历/释放、多项式或记录的组织。

你做这套编程题时，基本都能落到上面某几类的组合。

## 2. 统一做题流程（强烈建议每题都按这个顺序）

### 2.1 先把 I/O 写出来（先跑通，再优化）
1) 把输入全部读进来（先别急着算），确认读入次数和类型都正确。
2) 先用调试输出（或临时 `printf`）确认数据没读错，再写核心算法。
3) 最后删除调试输出，按题目要求格式化输出。

### 2.2 把题目转成“变量 + 约束 + 目标”
你要能回答三句话：
- 输入是什么？（数据类型、数量、分隔符、是否多组）
- 输出是什么？（格式、精度、分隔符、是否换行）
- 要计算什么？（目标函数/判定条件/构造结果）

### 2.3 选模板，而不是“现场发明”
常见模板（后面每个专题会展开）：
- 区间映射：`if/else if/else`
- 枚举因子：`for (i=1; i*i<=n; i++)`
- gcd：欧几里得
- 级数求和：循环 + 终止条件（精度/项数）
- 图形打印：行列坐标思想
- 进制转换：按位/按 8 位分组
- 字符串转整数：跳空白、读符号、累积
- 回溯：递归 + 合法性检测 + 剪枝
- 链表：节点结构 + 尾插/头插 + 遍历输出

## 3. 编程题最常见的“隐藏坑”
### 3.1 输入格式坑
- 输入里带逗号/点号等固定字符：格式串要匹配。
- 多组数据：要在外层加循环，注意每组输出换行。
- 字符串读入：是否允许空格？若允许要用 `fgets`（训练题若给 `gets`，理解其行为即可）。

### 3.2 精度与输出坑
- 小数保留位数：`%.1f`、`%.2f` 等要看清。
- 输出末尾是否需要换行/空格/点号。

### 3.3 边界坑
- `n=0/1` 这类最小值是否有定义？
- 负数、0、非常大时是否溢出（必要时用 `long long`）。
- 数组下标是否越界（尤其 `j+1`）。

## 4. 自检清单（提交前 1 分钟必做）
- 所有 `scanf` 的 `&` 都正确吗？字符串数组名没有乱加 `&` 吗？
- 循环边界是否能覆盖全部数据？不会多读/少读？
- 输出格式是否严格符合要求（精度、分隔符、换行）？
- 是否存在未初始化变量、未赋值指针、内存泄漏（链表题）？
