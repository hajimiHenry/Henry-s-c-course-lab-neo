# 编程题讲义：二进制与 IP 地址转换

> 目标：掌握“二进制字符串 → 十进制点分”的通用方法，并理解为什么按 8 位分组。

## 1. IP 地址的结构（只讲计算部分）
一个 IPv4 地址由 32 位组成，通常分成 4 组，每组 8 位：
```
aaaaaaaa bbbbbbbb cccccccc dddddddd
```
每 8 位对应一个 0~255 的十进制数，最终输出为：
```
A.B.C.D
```

## 2. 核心：8 位二进制转十进制
### 2.1 逐位累积法（最稳）
把一个 8 位二进制数 `s[0..7]` 转为十进制：
```c
value = 0;
for (k = 0; k < 8; k++) {
    value = value * 2 + (s[k] - '0');
}
```
这等价于“左移一位再加当前位”。

### 2.2 用位运算也可以（可选）
如果你已经把字符转成 0/1，也可以：
```c
value = (value << 1) | bit;
```

## 3. 32 位字符串的处理流程
1) 读入长度为 32 的字符串 `s`
2) 对 `i = 0..3`：
   - 取子串起点 `i*8`
   - 转成十进制 `A[i]`
3) 输出 `A[0].A[1].A[2].A[3]`

## 4. 输出格式
注意点分中间用 `.`，不是空格。可以写：
```c
printf("%d.%d.%d.%d\n", a0, a1, a2, a3);
```

## 5. 自检清单
- 输入是否保证 32 位？（若可能有换行，注意读入方式）
- 每一组是否刚好 8 位？下标是否写对？
- 字符 `'0'/'1'` 是否正确转换为 0/1？
